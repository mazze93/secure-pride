# Security Policy

## Reporting a Vulnerability

Do **not** open public GitHub issues for security vulnerabilities.

### Report to:

**security@securepride.org**

Please include:

- Affected component/version
- Vulnerability description
- Reproduction steps (if applicable)
- Potential impact
- Suggested fix (optional)

We commit to:

- Initial response within 5 business days
- Good-faith security review
- Responsible disclosure timeline
- Credit (if desired)

## Scope

**In-scope:** Secure Pride infrastructure, tooling, and documentation  
**Out-of-scope:** Third-party dependencies (please report to upstream maintainers)

## Security Standards

All contributions must:

- Have zero telemetry or hidden data collection
- Protect sensitive identity data
- Pass security validation checks
- Be reviewed before merge
